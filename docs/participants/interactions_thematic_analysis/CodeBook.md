**CODES**

**User Naive Prompt**: A simple prompt used by the user, such as zero-shot, without context.

**User Statement Prompt:** The situation in which the user copies the task description verbatim and includes it directly in the prompt. 

**User Provides Context**: The user provides context about the task for the model (e.g., system description). 

**User Defines Template**: The user defines the template (i.e., output format) for the task.

**User Contextualized Prompt**: An enhanced prompt used by the user with context information (e.g., specific questions).

**User Asks Improvements**: The user asks the AI model to improve its solution, such as a title or adherence to the template. It may mean critical thinking (learning).

**User Acquiring Knowledge:** An interaction in which the user asks the LLM for technical assistance

**User Repeat Ask:** The user repeats the same or a very similar question.

**User out of Context**: This label was used when the user provides unrelated information or prompts the Model for unrelated tasks.

**User Instruct Model:** An interaction in which the user provides explicit instructions to the model on how to perform

**Model Confirm Instructions:** The Model restates the instructions, serving as a support mechanism to ensure correct interpretation.

**Model Explanation**: The AI model explains a concept to the user. It may mean teaching.

**Model Asks for Information**: The AI model asks for further information to complete the task.

**Model Adequate Solution**: The AI model outputs a hopeful solution (response) to the task.

**Model Inadequate Solution**: The AI model generates a response that fails to adequately address the task requirements.

**Model Offers Reinforcements**: The AI model offers more concepts or examples to the user. It may mean teaching.

**Model out of Context**: Used when the Model provides unrelated information.
